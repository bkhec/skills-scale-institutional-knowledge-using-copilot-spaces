# OctoAcme — Team Onboarding Checklist

## Purpose
Streamline onboarding for new team members joining OctoAcme projects, ensuring they understand their role, responsibilities, and how to collaborate effectively with other roles.

## General Onboarding (All Roles)

### First Day
- [ ] Access to company systems (email, Slack, GitHub, etc.)
- [ ] Introduction to team members and their roles
- [ ] Review org chart and reporting structure
- [ ] Overview of current projects and priorities
- [ ] Access to project documentation (this docs/ folder)

### First Week
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) document
- [ ] Understand communication cadence and meeting schedules
- [ ] Set up development environment (if applicable)
- [ ] Shadow key meetings (standups, planning, etc.)
- [ ] Meet 1-on-1 with key collaborators from other roles

### First Month
- [ ] Participate in at least one full project cycle
- [ ] Contribute to team retrospective
- [ ] Complete role-specific onboarding tasks (see below)
- [ ] Establish regular communication patterns with cross-functional partners

---

## Role-Specific Onboarding

### Developer Onboarding
- [ ] Clone repositories and run local builds
- [ ] Review coding standards and PR guidelines
- [ ] Set up local development environment and tools
- [ ] Review CI/CD pipeline with DevOps Engineer
- [ ] Complete first small bug fix or feature
- [ ] Participate in code review process
- [ ] Understand testing strategy and requirements
- [ ] Learn deployment process and environments

### UX Designer Onboarding
- [ ] Access design tools (Figma, Sketch, etc.)
- [ ] Review design system and component library
- [ ] Understand user research processes and schedule
- [ ] Review existing user personas and journey maps
- [ ] Meet with Product Manager to discuss upcoming features
- [ ] Shadow usability testing sessions
- [ ] Learn design review and approval process
- [ ] Understand how designs are handed off to developers

### DevOps Engineer Onboarding
- [ ] Access to infrastructure and deployment systems
- [ ] Review architecture diagrams and infrastructure docs
- [ ] Understand CI/CD pipeline configuration
- [ ] Review monitoring, alerting, and logging systems
- [ ] Learn incident response procedures
- [ ] Understand backup and disaster recovery processes
- [ ] Review security and compliance requirements
- [ ] Shadow a deployment to production

### Data Analyst Onboarding
- [ ] Access to data platforms and analytics tools
- [ ] Review existing dashboards and reports
- [ ] Understand data sources and data models
- [ ] Learn metric definitions and KPIs
- [ ] Review data quality and validation processes
- [ ] Meet with Product Manager to understand business questions
- [ ] Learn experimentation and A/B testing framework
- [ ] Understand data privacy and compliance requirements

### Customer Success Onboarding
- [ ] Access to CRM and support ticketing systems
- [ ] Review customer segments and key accounts
- [ ] Understand product features and roadmap
- [ ] Learn escalation procedures
- [ ] Shadow customer calls and support interactions
- [ ] Review customer feedback collection processes
- [ ] Understand how to communicate with Product/Project teams
- [ ] Learn customer success metrics and goals

### Product Manager Onboarding
- [ ] Review product strategy and roadmap
- [ ] Understand customer segments and personas
- [ ] Access to product analytics and data tools
- [ ] Review backlog and prioritization framework
- [ ] Meet with key stakeholders and leadership
- [ ] Understand go-to-market and release processes
- [ ] Learn how to write requirements and acceptance criteria
- [ ] Review competitive landscape and market research

### Project Manager Onboarding
- [ ] Review current project portfolio and status
- [ ] Understand project management tools and processes
- [ ] Learn risk management and escalation procedures
- [ ] Access to project boards and tracking systems
- [ ] Review meeting facilitation best practices
- [ ] Understand stakeholder communication cadence
- [ ] Learn resource planning and capacity management
- [ ] Shadow experienced PM on project coordination

### Scrum Master Onboarding
- [ ] Review team's current agile practices
- [ ] Understand team velocity and historical metrics
- [ ] Learn how to facilitate scrum ceremonies
- [ ] Access to sprint planning and tracking tools
- [ ] Meet with team members individually
- [ ] Review impediment log and resolution process
- [ ] Understand how to coach team on agile principles
- [ ] Learn escalation paths for blockers

## Cross-Functional Collaboration

### Key Collaboration Patterns to Learn
All team members should understand these collaboration touchpoints:

**Developer ↔ UX Designer**
- Design review and feasibility discussions
- Implementation questions and clarifications
- Usability issue identification and resolution

**Developer ↔ DevOps Engineer**
- Deployment coordination and troubleshooting
- Infrastructure needs and environment setup
- Performance and scalability discussions

**Product Manager ↔ Data Analyst**
- Metric definition and success criteria
- Experiment design and results analysis
- Data-driven prioritization decisions

**Project Manager ↔ All Roles**
- Status updates and progress tracking
- Risk identification and mitigation
- Timeline and dependency management

**Customer Success ↔ Product/Project Teams**
- Customer feedback and feature requests
- Bug prioritization and customer impact
- Release planning and change communication

## Ongoing Learning & Development
- [ ] Subscribe to team communication channels
- [ ] Bookmark key documentation and resources
- [ ] Set up regular 1-on-1s with manager and key collaborators
- [ ] Join relevant communities of practice
- [ ] Identify mentors for specific skills or domains
- [ ] Contribute to documentation improvements
- [ ] Participate in knowledge sharing sessions

## Checklist Completion
Once onboarding is complete:
- [ ] Review checklist with manager
- [ ] Provide feedback on onboarding experience
- [ ] Suggest improvements to onboarding process
- [ ] Transition from onboarding to regular team contributions
